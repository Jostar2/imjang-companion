# ADR-001

## Title

Real-estate field visit MVP as a mobile-first web application with structured visit sessions

## Decision

Use a Next.js frontend and FastAPI backend with PostgreSQL for domain data and S3-compatible storage for images.

Model the domain around:

- Project
- Property
- Visit Session
- Checklist Response
- Score Summary
- Attachment
- Report Summary

## Why this decision

- mobile-first web is sufficient for MVP without native app overhead
- visit session is the core aggregate that groups checklist responses, notes, scores, and attachments
- manual property entry keeps the first release bounded
- object storage keeps binary attachments out of the main relational store

## Rejected alternatives

- native mobile app first
  - rejected because it increases delivery complexity too early
- spreadsheet export first with no app workflow
  - rejected because field capture structure is the core problem to solve
- external listing integration first
  - rejected because it expands scope and adds unstable dependencies

## Service boundaries

- frontend app
  - project/property list
  - visit checklist flow
  - comparison and report views

- backend API
  - auth/session
  - project/property CRUD
  - visit session CRUD
  - checklist response persistence
  - score aggregation
  - attachment metadata
  - report generation payload

## Data model sketch

- users
- projects
- properties
- visits
- checklist_sections
- checklist_responses
- visit_scores
- attachments
- reports

## Key invariants

- a visit belongs to exactly one property
- a property belongs to exactly one project
- required checklist sections must be complete before a visit is closed
- attachments must be mapped to the correct visit id
- comparison scores must be reproducible from stored responses

## Release strategy

- start with manual property entry only
- stage attachment upload early because it is a technical risk
- treat comparison and report as separate bounded UI slices
- keep production approval manual

## Rollback direction

- if attachment flow causes failures, disable uploads and keep notes/scores path
- if comparison/report breaks, preserve visit capture flows and hide the summary view
