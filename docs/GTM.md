# GTM.md

## Positioning

Do not position this as a general real-estate app.

Position it as:

- a buyer-side due diligence workspace
- a field-visit comparison and reporting system
- a replacement for scattered notes, gallery photos, and spreadsheets
- a buyer-agent-first workflow for repeated visit reporting

## Priority channels

### 1. Founder-led outbound

Target:

- independent buyer agents first

Secondary after the first buyer-agent conversations:

- small investor operators
- diligence-heavy property scouts

Message:

- stop losing visit context
- deliver same-day structured reports
- compare properties with stored rationale, not memory

### 2. Content and search

Topics:

- imjang checklist templates
- property comparison frameworks
- field visit red-flag guides
- report examples for buyers and investors

Purpose:

- capture search language and objections, not the first revenue
- collect language that sharpens the ICP
- build credibility for outbound outreach

### 3. Partner distribution

Potential partners:

- buyer consultants
- home inspection professionals
- renovation or repair reviewers

This is a later channel, not the first one.

## Launch funnel

1. founder target list
2. outbound message
3. 20-minute workflow review
4. 2-week paid pilot
5. activated workspace
6. same-day report usage
7. `Solo Pro` conversion

## Key messages

- "Turn field visits into structured decisions."
- "Compare properties from evidence, not memory."
- "Capture once in the field, deliver a report the same day."
