# Landing Page Copy

## Goal

Use this page to convert a warm outbound lead or referral into one of two actions:

- `Request pilot`
- `Book a 20-minute workflow review`

The page should not try to explain the whole product. It should answer three questions fast:

1. What pain does this remove?
2. Who is it for?
3. What happens next if I reply?

## Hero

- Headline: Turn field visits into structured decisions.
- Subheadline: Capture notes, scores, photos, and red flags in one mobile workflow, then compare properties from evidence instead of memory.
- Primary CTA: Request pilot
- Secondary CTA: Book workflow review
- Support line: Built for repeated-use operators, not one-off listing browsing.

## Section 1: Problem

- Title: Field diligence breaks down after the visit.
- Body:
  Teams doing repeated site visits usually split evidence across chat threads, photo rolls, and spreadsheets. By the time they review candidates, the reasoning behind each note is incomplete or missing.
- Bullets:
  - notes and photos are scattered across tools
  - red flags are remembered inconsistently
  - comparison happens from memory instead of structured evidence
  - same-day report delivery turns into manual cleanup work

## Section 2: Product promise

- Title: One workflow from visit capture to report delivery.
- Body:
  Imjang Companion is a buyer-side diligence workspace. It does not replace listing discovery. It makes the visit itself structured, comparable, and reviewable.
- Feature proof:
  - capture required property, building, and neighborhood sections on mobile
  - autosave and restore in-progress visit drafts in the same browser
  - attach photos without losing the rest of the visit if upload retry is needed
  - compare candidate properties inside one project
  - open a report-ready summary from the latest completed visit

## Section 3: Persona variant A

- Persona: Buyer agent / repeated-use operator
- Headline: Deliver same-day visit reports without stitching together chat, camera roll, and spreadsheets.
- Proof bullets:
  - mobile checklist during the visit
  - faster comparison across candidate properties
  - report-ready structure after completion
- CTA copy:
  - Primary: Request buyer-agent pilot
  - Secondary: See sample workflow

## Section 4: Persona variant B

- Persona: Small investor team
- Headline: Standardize diligence notes across repeated visits and keep the rationale behind each recommendation.
- Proof bullets:
  - structured field capture
  - repeatable checklist categories
  - faster internal review packet after each visit
- CTA copy:
  - Primary: Request investor-team pilot
  - Secondary: Review onboarding path

## Section 5: Pilot offer block

- Title: Pilot structure
- Body:
  We run a founder-led pilot with teams already doing repeated visits. The goal is to replace the current notes-plus-photo workflow for a small number of live visits, then measure whether comparison and report turnaround improves.
- Offer bullets:
  - 2-week guided pilot
  - onboarding help for the first project
  - direct feedback loop on missing workflow steps
  - pricing discussion only after real usage review

## FAQ / objection handling

- "Is this a listing portal?"
  - No. It is a due-diligence workspace after you already have real candidates to visit.
- "Does it work offline?"
  - V1 is online-first. It restores in-progress drafts in the same browser, but it does not queue offline writes.
- "Who is it for?"
  - Repeated-use operators first: buyer agents, investor operators, and diligence-heavy property scouts.
- "What do I need to start?"
  - One active project, at least two candidate properties, and one real visit you want to compare or summarize.

Use this as the short outbound blurb in email, DM, or community follow-up:

> We are building a mobile field-diligence workflow for teams that visit multiple properties and then lose context across notes, photos, and spreadsheets. If you already run repeated visits, I can set up a small pilot and show you how the visit-to-report flow works on a real project.
